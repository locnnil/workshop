Title: Implement GPU-Accelerated 3D Rubik's Cube Renderer

Mission:
Read planning documentation from ../design/ directory and implement a
GPU-accelerated 3D Rubik's cube renderer with continuous rotation and
realistic lighting. Produce working code that displays a smoothly
rotating, properly lit 3D cube with FPS counter, following the
architecture and technology choices specified in the design documents.

Context:
- Planning documents exist in ../design/ (tech-stack.md, design.md, requirements.md)
- Source specification: ./prompts/rubik-shared.md
- Target platform: Ubuntu 24.04, Python 3.10+
- Purpose: Pure rendering demonstration with GPU acceleration
- Implementation must be complete and immediately runnable
- If planning documents have gaps or ambiguities, fill them with reasonable defaults

Task Requirements:
1. Read and parse all planning documents from ../design/
2. Implement complete architecture:
   - 3x3x3 Rubik's cube geometry with visible gaps between cubies
   - GPU-accelerated rendering using PyOpenGL
   - Phong or Blinn-Phong lighting model
   - Continuous multi-axis rotation with randomized speeds
   - FPS counter overlay
3. Create dependency management (pip requirements.txt or as specified)
4. Provide README.md with clear usage instructions
5. Ensure code follows modern Python best practices

Process / Steps:
1. Read planning documents:
   - Load ../design/tech-stack.md to identify required libraries
   - Load ../design/design.md to understand architecture and module structure
   - Load ../design/requirements.md to capture functional requirements
   - If files don't exist or are incomplete, STOP and report the issue
2. Set up project structure:
   - Create appropriate file layout based on design.md
   - Create requirements.txt based on tech-stack.md
   - Must use Pygame + PyOpenGL (per rubik-shared.md constraints)
3. Implement core modules (in order):
   a. Cube geometry generation:
   - Generate vertex data for 26 visible cubies (3x3x3 with hollow center)
   - Apply gaps between cubies (scale to ~0.9 of unit size)
   - Assign distinct colors to each face (white, yellow, red, orange, green, blue)
   - Position cubies correctly in 3D space
   b. Lighting and materials:
   - Implement Phong or Blinn-Phong lighting setup
   - Configure positional light source
   - Define material properties (ambient, diffuse, specular)
   - Enable smooth shading
   c. Rendering loop:
   - Initialize Pygame window (640x480, windowed)
   - Set up OpenGL context with perspective projection
   - Enable depth testing and backface culling
   - Implement continuous rotation on X, Y, Z axes
   - Randomize rotation speeds at initialization (15-45 deg/sec per axis)
   - Calculate and display FPS counter (upper-left corner)
   d. Scrambling:
   - Randomize initial cube state (scrambled appearance)
   - Apply randomization only at initialization
   e. Main entry point:
   - Initialize all systems
   - Run render loop until ESC or window close
   - Clean shutdown
4. Create README.md:
   - Project description and purpose
   - Installation instructions (pip install -r requirements.txt)
   - Usage instructions (python3 rubiks_cube_renderer.py)
   - Dependencies list
   - Requirements (Python version, platform, GPU)
5. Verify implementation:
   - Test that all dependencies install cleanly
   - Execute renderer and verify visual output
   - Confirm smooth rotation at 60+ FPS
   - Verify lighting effects are visible
   - Test window close mechanisms (ESC and X button)
