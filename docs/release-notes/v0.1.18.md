## What's new in Workshop v0.1.18

This release improves debugging for SDK hooks, addresses LXD profile update
issues, and introduces networking improvements and initial reference
architecture documentation.

### Verbosity mode for debugging SDK hooks

A new verbosity mode makes it easier to debug SDK hooks when using `launch`,
`refresh`, or `sketch-sdk`. You can now run these commands with `--verbose` and
`--wait-on-error`, which stops on error and lets you shell into the Workshop
container for troubleshooting.

### Improved handling of LXD profile updates

Workshop now implements transactional handling of LXD profile updates. If an
update fails (such as from a port conflict), the profile is rolled back to a
previous working version, avoiding broken states.

### Workshop networking improvements

Workshop now maintains a stable IP across restarts, improving reliability for
services that rely on consistent networking.

## SDK updates

Our current focus is to help teams build a solid set of reference SDKs and
demonstrate each with a reference project.

### OpenVINO SDK

The Intel OpenVINO SDK is now available, reusing existing Snap work:

```yaml
name: dev
base: ubuntu@24.04
sdks:
  - name: openvino-toolkit
    channel: noble/edge
```
