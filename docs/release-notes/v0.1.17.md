# Workshop v0.1.17 release notes

## 10 June 2025

These release notes cover new features and changes in Workshop v0.1.17.

## Requirements and compatibility

Workshop relies on Snap and LXD:

- See the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/)
  for setup instructions.
- Refer to the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/)
  for development prerequisites.

## What’s new in Workshop v0.1.17

This release improves flexibility in hook execution, adds a new how-to guide for
VS Code integration, and fixes compatibility with non-standard usernames in LXD
projects.

### New `setup-project` hook for better caching and layout control

A new `setup-project` hook now runs after interfaces are connected and all SDKs
are installed. This improves cache utilization (e.g. `pip`, `apt`) and ensures
that project directories do not affect post-setup snapshots.

Some related improvements were made: Workshop now runs state hooks for all SDKs
retained in a refreshed workshop; hook scripts are executed without enforcing
`nounset`, avoiding issues with some environment scripts; volume attachments no
longer alter root directory permissions; SDKs are mounted read-only; non-test
code now uses `dirs.XdgRuntimeDirBase` instead of hardcoded `/run/user` paths;
and `cloud-init` configuration has moved to the `apt` module rather than custom
files.

**Note**: SDKs that use `/project` during `setup-base` may break with this
update. The ROS 2 SDK is known to be affected.

### Visual Studio Code integration guide

A new guide explains how to use Workshop with VS Code, whether through a remote
SSH session or via the browser. It’s now linked from the main tutorial and
visible in the documentation sidebar.

### Improved support for non-POSIX usernames in LXD

Usernames that include characters like `@` previously interfered with LXD
snapshot logic, where `@` is interpreted as a snapshot delimiter. To fix this,
Workshop now ensures that only alphanumeric characters are used in LXD project
names. If the username contains unsupported characters, the user’s numeric UID
is used instead.
