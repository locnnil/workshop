# Workshop v0.1.30 release notes

## 29 April 2026

These release notes cover new features and changes in Workshop v0.1.30.

## Requirements and compatibility

Workshop relies on Snap and LXD:

- See the [Tutorial](https://ubuntu.com/workshop/docs/tutorial/) for setup instructions.
- Refer to the [Contribution Guide](https://ubuntu.com/workshop/docs/contributing/) for development prerequisites.

## What's new in Workshop v0.1.30

This release introduces SDK Store integration, a new `workshop restore`
command, redesigned CLI output, WSL support, and numerous reliability and
code-quality improvements.

### SDK Store integration

Workshop now resolves, downloads, and queries SDKs through the new SDK Store,
replacing the previous GCS-based backend which has been removed entirely. A
store client imported from the Juju ecosystem handles channel resolution using
logic based on snapd's model. The `sdk find` command lets users search the
store by query string, and `sdk info` now queries the store directly with
optional filtering by base or architecture.

### Workshop restore command

A new `workshop restore` command resets a workshop to a known-good snapshot
from the latest refresh or launch change. `workshop restore` replaces
`workshop refresh --restore` option.

### CLI usability overhaul

- Redesigned table output with a `--no-headers` flag for scripting.
- Mistyped commands now suggest the closest match instead of printing full help.
- Commands scoped to a project no longer repeat the project column in tables.
- Shell completion installed for the `sdk` CLI, with best-effort project and
  string-flag completions for `workshop` commands.
- SDKs sorted by name then revision in `sdk list`.
- `sdk info` prints all project paths.
- `--verbose` sets `-x` mode for all SDK hooks.

### WSL compatibility

A btrfs storage fallback was added for WSL environments where other backends
are unavailable, broadening Workshop's platform support.

### LXD backend improvements

SDK imports now use the `import_custom_volume_tar` API with graceful
interruption handling. The minimum LXD version has been raised to 6.6. The Go
LXD client library has been updated for forward compatibility, including
`CopyInstance` and `DeleteProject` API changes. Instance start and stop logic
has been simplified.

### Bug fixes

- Fixed duplicate auto-connections.
- Fixed connection teardown for cross-workshop interfaces and when Setup fails.
- Fixed systemd environment lookup for root users.
- Xauthority units are now created even if migration failed and only activated
  when the desktop interface is connected.
- Systemd unit installation made idempotent.
