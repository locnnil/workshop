# Workshop v0.1.13 release notes

## 28 March 2025

These release notes cover new features and changes in Workshop v0.1.13.

## Requirements and compatibility

Normally, Workshop relies on Snap and LXD in its operation:

- For installation details, see the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/).
- For additional development requirements, see the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/).

Note that the required LXD version was increased to 6.3 with this release.

## What's new in Workshop v0.1.13

This update improves refresh performance, enhances error messaging and updates
device handling.

### Better performance at refresh with snapshots

The refresh resolver now intelligently restores workshops from snapshots to
speed up refresh operations. Snapshots are selected based on SDK updates and
changes in workshop definitions. Workshops with updated base images rebuild from
scratch using LXD's RebuildInstance API. The local sketch SDK refreshes every
time due to update-detection limitations.

**Breaking changes**: Storage pools now auto-remove previous snapshots upon
restore (`volume.zfs.remove_snapshots: "true"`). The `system` SDK is treated
like other installed SDKs in workshop operations.

**Implementation notes**: Instances are now stashed using `CopyInstance` instead
of migrating, allowing safer rebuilds and restores.

**Known issues**: Unable to copy instances with snapshots across projects ([LXD
issue #15078](https://github.com/canonical/lxd/issues/15078)).

### Enhanced `--wait-on-error` error messages

This release clarifies error messages during refresh or launch failures by
displaying the latest relevant error log line and suggesting actionable next
steps directly within the CLI output. Stored hook logs are limited to the most
recent nine lines for efficiency:

```
ERROR cp: cannot stat './not-found': No such file or directory (exit code: 1)
error: cannot complete refresh for "dev", execution is paused

To proceed, resolve the issue and run 'workshop refresh --continue dev'
To cancel and undo: 'workshop refresh --abort dev'
To view more information: 'workshop tasks 1077'
```

### Camera interface migrated to unix-hotplug

Workshop now uses the new `unix-hotplug` subsystem introduced in LXD 6.3,
supporting dynamic media devices with automatic device ownership matching the
host system.

### Doc and code coverage

- Migrated linting to `golangci-lint` 2.0.
- Updated `golangci-lint` pre-commit hooks.
- Documentation now has more details on the use of interfaces and the system
  SDK.
