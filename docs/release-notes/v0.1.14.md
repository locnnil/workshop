# Workshop v0.1.14 release notes

## 11 April 2025

These release notes cover new features and changes in Workshop v0.1.14.

## Requirements and compatibility

Normally, Workshop relies on Snap and LXD in its operation:

- For installation details, see the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/).
- For additional development requirements, see the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/).

## What's new in Workshop v0.1.14

Apart from some behind-the-scenes changes, this update mostly focuses on
improving the CLI experience.

### Better subcommand grouping in `workshop --help`

The output produced previously could be confusing; this time, we're trying to
present a more coherent picture of our many subcommands and their purpose.

### The `--no-wait` option is now available with refresh

Now, `workshop refresh` supports the `--no-wait` option to immediately return
the ID of the change rather than wait indefinitely.

### Improved error messaging when LXD appears unavailable

The previous error message when LXD was not installed was unclear and made it
difficult for users to diagnose the problem. The `workshopd` daemon now detects
the absence of the LXD socket and displays a user-friendly message informing
users that LXD is not installed or running, with guidance on how to proceed.

### Doc and code coverage

- The starter pack was updated to the [latest
  version](https://github.com/canonical/sphinx-docs-starter-pack/releases/tag/1.0.1).
- The tutorial was revamped to improve navigation experience.
