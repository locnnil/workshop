## What’s New in Workshop v0.1.11

This is mostly a convenience release, following an intermediate bugfix release a
week ago, but there's an important change in how we handle workshop states.

### Added a new `Waiting` status; `Pending` less visible now

This release improves handling of paused states, ensuring refresh operations
don’t proceed when another sketch or `--wait-on-error` is active. Documentation
now consistently refers to this state as `Waiting` or `Paused`, and additional
testing prevents regressions.

It also removes unintended status changes during `exec`, enabling previously
blocked operations in shell sessions. A warning now appears when connecting or
disconnecting interfaces in an active shell, as changes won’t take effect
immediately.

### Doc and Code Coverage
