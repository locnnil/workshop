# Workshop v0.1.12 release notes

## 21 March 2025

These release notes cover new features and changes in Workshop v0.1.12.

## Requirements and compatibility

Normally, Workshop relies on Snap and LXD in its operation:

- For installation details, see the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/).
- For additional development requirements, see the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/).

## What’s new in Workshop v0.1.12

This update reorganizes Workshop’s data structure, improves command-line
tooling, and introduces networking enhancements for more flexible use cases.

### New path for Workshop user data

Workshop content now respects `XDG_DATA_HOME` (defaulting to `~/.local/share` if
that variable isn’t set). Additionally:

- Projects are moved into unique directories per workshop.
- Content paths have been renamed to better reflect their function,
  consolidating mounts and sketches in clearer locations.
- Mount-path queries now rely on `systemd`, which requires lingering to be
  enabled for certain users during testing.

**Breaking change**: Because sketch and mount directories have shifted, existing
workshops must be removed before adopting the new layout. If you’re using the
snap, these changes are made automatically.

### Shell completion for Workshop scripts

Shell completion is now supported for more Workshop commands, specifically
`shell`, `run` and `scripts`:

- Fixes multiple-completion issues and prevents extraneous warnings.
- Improves handling of relative `--project` paths.
- Includes a workaround for a known [Cobra framework
  issue](https://github.com/spf13/cobra/issues/1877).
- When debugging shell completion, set `BASH_COMP_DEBUG_FILE=/tmp/err.log`.

### Now `workshop tasks` defaults to the latest change

When you run `workshop tasks` for a project without specifying a change ID, the
command automatically uses the most recent change associated with that project.

### New tunnel interface

A `tunnel` interface enables communication between services in the workshop and
on the host:

- It adds extra details to the `workshop info` API and is included in the
  `sketch-sdk` template.
- Interfaces without defined plug and slot sanitizing functions cannot have
  attributes, preventing unexpected behavior.
- Testing this interface requires `netcat` and `systemd` version 230+; most
  modern distributions already meet these requirements.

**Known issues**:

- If a connect operation fails, the LXD profile may still update, causing
  misleading success on subsequent retries (but safe to disconnect).
- Disconnecting certain socket plugs doesn’t remove the socket ([LXD issue
  #14972](https://github.com/canonical/lxd/issues/14972)).
- Error messages for port conflicts are inconsistent in some cases ([LXD issue
  #15023](https://github.com/canonical/lxd/issues/15023)).
- Proxy devices can close prematurely ([LXD issue
  #15024](https://github.com/canonical/lxd/issues/15024)).
- Currently, `workshop info` uses active connections instead of profiles. This
  is a temporary measure to handle hidden system SDKs.

### Doc and code coverage

- Introduced a packaged CLI documentation generator, **gencodo**
  ([repo](https://github.com/canonical/gencodo)).
- Moved certain SDKcraft documentation tests to this repository for consistent
  coverage.
- Added a new release checklist to the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/).
