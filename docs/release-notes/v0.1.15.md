# Workshop v0.1.15 release notes

## 29 April 2025

These release notes cover new features and changes in Workshop v0.1.15.

## Requirements and compatibility

Workshop relies on Snap and LXD:

- See the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/)
  for setup instructions.
- Refer to the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/)
  for development prerequisites.

## What's new in Workshop v0.1.15

This update focuses on smarter SDK handling, improves local workflows, and adds
more control over your custom SDKs.

### Smarter handling of sketch SDKs

Sketch SDKs are now tracked on the host system, not inside workshops. They are
mounted read-only, improving reliability, especially when editing files. Also,
installing SDKs now triggers automatic snapshots for easier recovery.

### Project-defined SDKs are now supported

You can now define lightweight SDKs directly inside your project under
`.workshop/<name>/`. Workshops can reference these with the `project-` prefix.
This gives you full control without needing to publish or modify store SDKs.

### New eject option for sketch SDKs

The `sketch-sdk --eject` command lets you turn a temporary sketch SDK into a
permanent project SDK. This makes it easier to share, modify or version-control
your custom SDKs.
