# Workshop v0.1.19 release notes

## 4 July 2025

These release notes cover new features and changes in Workshop v0.1.19.

## Requirements and compatibility

Workshop relies on Snap and LXD:

- See the
  [Tutorial](https://canonical-workshop.readthedocs-hosted.com/stable/tutorial/)
  for setup instructions.
- Refer to the [Contribution
  Guide](https://canonical-workshop.readthedocs-hosted.com/stable/contributing/)
  for development prerequisites.

## What’s new in Workshop v0.1.19

This release adds support for symlinked and `zstd`-compressed SDKs, improves
Wayland compatibility for Snap apps, and expands internal documentation on
architecture and usage, refactoring some of its frequently used sections as
well.

### Support for symlinks and Zstandard-compressed SDKs

Workshop now handles SDKs that use Zstandard compression and contain symbolic
links, in preparation for upcoming changes in SDKcraft. Symlinks are preserved
in local SDKs to match behavior in Store-distributed ones.

### Wayland support for Snap applications

A compatibility fix improves how Wayland sockets are mounted in Workshop for
Snap-based applications like Firefox. Workshop now ensures the Wayland socket
path conforms to Snapd’s expected location pattern (`/run/user/*/wayland-*`),
allowing Snap confinement to access it. This resolves earlier issues where
symlinks to the Wayland socket were lost after a workshop restart.

### Improved documentation: starter pack SDK and architecture section, refactoring

Workshop now includes an in-project SDK to enable bootstrapping the
documentation starter pack within its own environment. A new documentation
section on internal architecture has also been added, with an overview of design
decisions; more expansions will follow soon.
