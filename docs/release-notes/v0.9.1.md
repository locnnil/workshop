# Workshop v0.9.1 release notes

## 8 June 2026

These release notes cover new features and changes in Workshop v0.9.1.

## Requirements and compatibility

Workshop relies on Snap and LXD:

- See the [Tutorial](https://ubuntu.com/workshop/docs/tutorial/) for setup instructions.
- Refer to the [Contribution Guide](https://ubuntu.com/workshop/docs/contributing/) for development prerequisites.

## What's new in Workshop v0.9.1

Workshop v0.9.1 adds a `custom-device` interface, improves SDK Store request
reliability, simplifies `refresh` errors, and improves how commands and SDK
metadata are shown to users.

### Custom-device interface

Workshop definitions can now declare `custom-device` plugs for SDKs that need
access to host devices grouped by subsystem:

```yaml
plugs:
  <NAME>:
    interface: custom-device
    subsystem: <SUBSYSTEM> # e.g. accel or input
```

### Unused SDK snapshot cleanup

Removing a workshop now also removes unused SDK snapshots that remain after the
workshop is gone. This fixes [issue#799](https://github.com/canonical/workshop/issues/799).

### User-facing workshop exec summaries

`workshop exec` changes and task summaries now show the command requested by the
user instead of Workshop's internal `sudo` execution wrapper.

### SDK Store retry logic

Workshop now retries failed pre-handshake requests to the SDK Store, improving resilience when
the store or network returns a transient error.

### SDK website output

`sdk info` now supports the `website` field for SDKs that publish website
metadata.

## What's new in SDKcraft

### SDK revision listing

`sdkcraft revisions` is a new command that lists SDK revisions available on the
store.

### Ubuntu 26.04 base support

SDKcraft now supports `ubuntu@26.04` as an SDK base.

## Reference SDK updates

The reference SDKs now include Ubuntu 26.04 base support. Publishing is in
progress, and the updated SDKs will be available soon.

Tests were added to the agentic and toolchain SDKs. We will continue improving
automatic end-to-end tests for the reference SDKs.
